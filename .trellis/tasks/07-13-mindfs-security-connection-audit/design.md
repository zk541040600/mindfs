# 设计：MindFS 安全与连接审计修复

## 边界

本变更覆盖浏览器 WebSocket 生命周期、现有 E2EE 会话管理、受保护 HTTP 响应和 E2EE 配置文件权限。HTTP 请求证明字段保持不变；通过已有握手中的 `proto_version` 协商 v2，不替换加密算法或引入新服务。

## 连接生命周期

`SessionService.disconnect()` 是有意停止的边界。它将清除重连计时器、探测状态和 socket，并把当前根目录置空。异步 `openSocket()` 在 await 后已检查活动根目录，因此无法在断开后的空目标状态创建 socket；下一次显式 `connect(rootId)` 会重新建立唯一连接。

## E2EE 重放控制

`e2ee.Manager` 拥有短生命周期的已消费证明/握手 nonce 记录和会话私有的 v2 帧计数器，避免把安全状态分散到 HTTP 与 WebSocket 层。

- 仅在 HMAC、时间窗和会话状态均成功验证后消费请求证明。
- 消费键包含用途、客户端 ID 与已验证值；条目在其安全窗口结束后清理。重复值拒绝，不触碰会话的 `LastSeenAt`。
- 握手在完整验证和密钥派生后消费 `client_nonce`，然后才替换同一客户端的会话。nonce 采用短生命周期记录，正常新握手的随机 nonce 不受影响。

v2 将协议版本写入 HKDF 输入和 server-accept proof，避免降级得到同一把传输密钥。v2 WebSocket 在现有 AES-GCM 密文内部封装 `{sequence,message}`，客户端与服务端各自维护一个递增方向计数；重复帧在业务处理前拒绝。服务端仍处理 v1 会话，保留已有发布客户端的帧格式。

HTTP 证明报文保持不变，但 v2 响应使用它作为 AES-GCM AAD：普通 JSON 是 `proof`，raw-file 是 `proof + "\\x1f" + contentType`。这使响应密文只能认证为其原始请求的结果，不需要新增按密文去重的全局缓存。浏览器在请求到响应之间保留五分钟、一次性的 proof-to-session 关联；未关联的加密响应直接失败。

明文 E2EE 错误并不携带配对码知识证明，因此只允许失效当前会话，不自动擦除保存的配对码。用户显式输入新码才会覆盖旧值。

## 文件系统根边界

`RootInfo` 在实际文件操作前先完成既有词法相对路径检查，再由同一内部校验逐段 `Lstat`/`EvalSymlinks` 已存在组件。解析目标必须继续位于解析后的根目录内；不存在的后缀只在全部已有父目录通过校验后才允许创建。这样保留根内符号链接，同时拒绝根内链接跳转到根外的读取、下载、上传和 watcher 注册。

`.mindfs` 是受管根内的专用元数据子树。其路径单独拒绝绝对值和 `..`，并复用根边界检查，避免 metadata API 因路径归一化折叠而访问根内其他文件或沿 `.mindfs` 软链写出根目录。

## 配置权限

`EnsureConfigAtPath` 保留“内容变化时原子写入”的路径；当内容未变时，额外对目标文件执行 `chmod(0600)`。权限收紧失败直接返回错误，避免将可读密钥配置误报为已确保安全。

## 回滚

所有变更为内存生命周期或权限收紧：回滚代码即可恢复旧行为。已经收紧的文件权限无需放宽；不会迁移或修改配置内容格式。
